How to Run Code:

We found the best matrix size for comparison testing to be 3840, so this is the recommended size to compare runtimes. If a different matrix size is selected, keep in mind that it must be a multiple of 48 for the mpi code.
*We've written custom scripts to make it easier to compile and run our code. Sequential is used in the examples, but replace script names appropriately.

Give permission to custom scripts:
  • chmod 755 c_seq r_seq

Compile source code using:
  • ./c_seq

Run using:
  • ./r_seq <size> <nthreads(openMP only)> <print(optionl)>

*size = desired matrix size, nthreads = number of threads to use
*To change MPI ntasks, edit mpi_slurm.sh first
*1 = print, default 0 = not print, matrix must be 9x9 or smaller


Sequential

Download Sequential Code

Our sequential baseline in C++. This performs row reductions on matrix a, slowly turning it into matrix U, while storing the coefficient multiplies to fill in matrix L. LU decomposition on a matrix of size 1920 should take around 23 seconds, and size 3840 will take around 3 minutes.

Input arguments: <matrix size> <print>
See How to Run above to compile and run code.

Download OpenMP Code

Uses shared matrices and dynamic scheduling to perform the LU decomposition. Using 16 threads with a matrix size of 3840 this should take around 9 seconds, and with 48 threads should take around 5 seconds.

Input arguments: <matrix size> <thread count> <print>

MPI

Download MPI Code

Uses MPI_Bcast to send coefficient multipliers to processes. Also uses MPI_Send and MPI_Recv to send data portions of matrix a to processes. With a mtrix size of 3840, using ntasks=24 and tasks-per-node=24, this should take around 8 seconds. With ntasks=96 and tasks-per-node=24 this should run in under 3 seconds.

Input arguments: <matrix size> <ntasks> <tasks-per-node> <print>
See How to Run above to compile and run code.
